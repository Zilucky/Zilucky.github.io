FAIR

Step1: Identify Scenario Components

  • webserver

    • Loss Event Probability: The loss event probability depends on the specific environment and security measures. If the web server is located on an internal network and protected by appropriate security controls, the probability of a loss event may be low. However, if the server is directly facing the public network or lacks effective security measures, the probability of a loss event may be higher.PLC and HMI devices

  • PLC and HMI devices

    • Loss event probability: the loss event probability of a PLC device depends on the criticality of the system it controls, network accessibility, and security measures. loss event probability may be higher where the PLC device controls critical infrastructure and has serious vulnerabilities and is susceptible to attack.Database

  • Database

    • Loss event probability: the loss event probability of a database depends on the value of its sensitive data, the security configuration of the database, and access control measures. If the database stores sensitive customer information or important business data, and the database has serious vulnerabilities and is susceptible to attacks, the loss event probability may be higher.

Step2: Evaluate Loss Event Frequency (LEF)

1. Web servers:

    ● The specific loss magnitude can be evaluated based on business value, recovery time objective (RTO), business continuity requirements and other factors. If only the peripheral Web servers are breached, the impact level is medium and the expected loss is in the tens of millions.

2. PLC and HMI devices:

    ● The specific loss magnitude can be assessed based on the value of the critical equipment or system, the impact of downtime, recovery time objectives, and other factors. If the PLC and HMI equipment is attacked to the point where the equipment does not work properly, the impact tier is high and the expected loss is in the hundreds of millions.

3. Database:

    ● Specific loss level can be assessed based on the value of the data, the impact of data loss, the difficulty of data recovery and other factors. If there is a database information leakage or loss of impact level is very high, the expected loss of several billion.

Loss Event Frequency = Threat Event Frequency * Vulnerability = (Contact Frequency * probability of Action) + (Threat Capability * Resistance strength)[1]

For Web servers:

Loss Event Frequency = Threat Event Frequency * Vulnerability = (100 times * 30%) times * (35% * 70%) = 7.35 times

Step3: Evaluate probable Loss Magnitude (PLM)

Loss Magnitude = Primary Loss * Secondary Loss = Primary Loss + (Secondary Loss Event Frequency * Secondary Loss Magnitude)[1]

For web servers:

Loss Magnitude = Primary Loss + Secondary Loss = Primary Loss + (Probability of Secondary Loss * Magnitude of Secondary Loss) = 1000w + (25% * 300w) = 1075w

PLM: High ( More than 1000w )

For PLC and HMI devices:

Loss Event Frequency = Threat Event Frequency * Vulnerability = (200 events * 70%) * (60% * 30%) = 15.2

Loss Magnitude = Primary Loss + Secondary Loss = Primary Loss + (Probability of Secondary Loss * Magnitude of Secondary Loss) = 10,000w + (25% * 1000w) = 10250w

PLM: Severe ( More than 1000w )

Loss Magnitude = Primary Losses + Secondary Losses = Primary Losses + (Probability of Secondary Losses * Magnitude of Secondary Losses) = 30000w + (40% * 5000w) = 32000w

PLM: Severe

Step4: Derive and Articulate Risk

Risk = Loss Event Frequency * Loss Magnitude[1]
